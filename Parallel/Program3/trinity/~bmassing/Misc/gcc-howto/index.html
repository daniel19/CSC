Using the GNU Compilers on the CS Linux Machines

The most-recent configuration for our classroom/lab machines includes multiple versions of the GNU compilers:

  • The most-recent version (4.4.7) available from the standard repositories for the distribution we use. This is the version you get by default.
  • More-recent versions obtained directly from the GNU project's Web site, installed in /usr/local-gcc-*-install. Currently these include version 4.8.1 (the most stable version available in summer 2013) and version 4.9.1 (the most stable version available in summer 2014). The simplest way to access the most recent version of the latest of these compilers from the command line is to use the command

    module load gcc-latest

    to add the install directory to your command search path and set other environment variables correctly. If you always want to use this version, you can put the above line at the end of file .bashrc in your home directory. Be advised that if you log onto a machine running an older configuration, this may result in an error message at login time. You can avoid this by instead putting in the .bashrc file the following lines:

    if [ -f /etc/modulefiles/gcc-latest ]
    then
    module load gcc-latest
    fi

    If for some reason you prefer to specify the version rather than using the latest one, replace gcc-latest with a specific version. The command module avail will show you the available choices.

    Note: If you compile with one of the non-default compilers, you should be sure that when you run your programs they can find the associated libraries. Again the simplest way to do this is with the above-described module load command.
